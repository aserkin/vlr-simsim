# `vlr-simsim`

Minimal SGsAP VLR simulator for MME interoperability testing.

The program accepts SCTP connections from an MME and emulates a very small
subset of VLR behavior over the SGs interface. Its intended use is lab testing
of voice-centric UEs and CS Fallback when no real VLR is available.

The implementation is intentionally narrow. It does not try to be a complete
3GPP TS 29.118 stack. It only responds to a few MME-initiated procedures and
ignores everything else.

## Supported procedures

Incoming SGsAP requests currently handled:

- `SGsAP-LOCATION-UPDATE-REQUEST` (`0x09`)
- `SGsAP-EPS-DETACH-INDICATION` (`0x11`)
- `SGsAP-IMSI-DETACH-INDICATION` (`0x13`)
- `SGsAP-RESET-INDICATION` (`0x15`)

Responses generated by the simulator:

- `LOCATION-UPDATE-ACCEPT` when both `IMSI` and `LAI` IEs are present
- `LOCATION-UPDATE-REJECT` when the request is malformed or missing mandatory IEs
- `EPS-DETACH-ACK`
- `IMSI-DETACH-ACK`
- `RESET-ACK` with a configured VLR name

Unsupported SGsAP messages are ignored and counted in the statistics.

## Current behavior

The simulator parses SGsAP messages as a message-type octet followed by TLV IEs.

For `LOCATION-UPDATE-REQUEST`:

- the request is accepted only if both `IMSI` and `LAI` are present and well-formed
- the response echoes `IMSI` and `LAI`
- otherwise the simulator returns `LOCATION-UPDATE-REJECT`

For detach procedures:

- if `IMSI` is present, it is echoed back in the corresponding ACK
- if `IMSI` is missing or malformed, the simulator still sends a bare ACK and logs a warning

For reset:

- the simulator always answers with `RESET-ACK`
- the `VLR name` IE is included in the response

## Requirements

- Python 3
- SCTP support on the host
- Python `sctp` module available to the interpreter

Depending on the environment, you may also want `libsctp` or the platform SCTP
runtime installed.

## Usage

Run the simulator:

```bash
python3 vlr.py <listen-ip> <listen-port>
```

Example:

```bash
python3 vlr.py 10.0.0.29 29118
```

Optional arguments:

```text
--vlr-name NAME       VLR name returned in RESET ACK, default: vlr-simsim
--reject-cause N      Reject cause used in LOCATION UPDATE REJECT, default: 8
--log-level LEVEL     DEBUG, INFO, WARNING, ERROR
```

Example with explicit options:

```bash
python3 vlr.py 10.0.0.29 29118 --vlr-name lab-vlr --log-level DEBUG
```

Configure your MME to use this host and port as its SGs/VLR peer.

## Signals and statistics

The process supports:

- `SIGUSR1`: dump message counters to the log
- `SIGINT`: stop cleanly
- `SIGTERM`: stop cleanly

Tracked counters:

- Location Update Requests
- EPS Detach Indications
- IMSI Detach Indications
- Reset Indications
- Other unsupported SGsAP messages

## Example systemd unit

```ini
[Unit]
Description=Lab SGsAP VLR simulator
After=network.target
StartLimitIntervalSec=0

[Service]
Type=simple
Restart=always
RestartSec=10
User=user
ExecStart=/usr/bin/python3 /home/user/dev/vlr-simsim/vlr.py 10.0.0.29 29118 --vlr-name lab-vlr

[Install]
WantedBy=multi-user.target
```

## Limitations

- This is not a full TS 29.118 implementation.
- Only a small set of SGsAP procedures is handled.
- No persistent subscriber state is maintained.
- No validation beyond basic IE boundary checks and mandatory IE presence is performed.
- Unsupported or vendor-specific procedures are ignored.

## Suggested next improvements

If you want to extend this simulator further, the most useful next steps are:

- add per-UE state keyed by IMSI
- make accept/reject behavior configurable per procedure
- add optional packet hex dumps for troubleshooting
- add automated tests with canned SGsAP frames
- split protocol parsing from socket handling to simplify future procedure work
